# データ設計メモ

## 基本方針

このアプリのデータは、最初は静的JSONまたはTypeScript配列で管理する。  
将来的にAPI化できるよう、型を明確に分ける。

## 自治体

```ts
type Municipality = {
  id: string;
  prefecture: string;
  name: string;
  municipalityCode?: string;
  links: SourceLink[];
  notes?: string;
  lastCheckedAt?: string;
};
```

## リンク

```ts
type SourceType =
  | "official"
  | "public"
  | "infrastructure"
  | "media"
  | "private"
  | "placeholder";

type SourceLink = {
  id: string;
  label: string;
  url: string;
  sourceName?: string;
  sourceType: SourceType;
  trustRank: "A" | "B" | "C" | "D";
  isPlaceholder?: boolean;
  lastCheckedAt?: string;
};
```

## 登録エリア

```ts
type SavedAreaRole = "home" | "work" | "familyHome" | "frequent";

type SavedArea = {
  role: SavedAreaRole;
  municipalityId: string;
  label?: string;
  createdAt: string;
  updatedAt: string;
};
```

## 防災資料

```ts
type LibraryCategory =
  | "earthquake"
  | "tsunami"
  | "flood"
  | "landslide"
  | "lifeline"
  | "stockpile"
  | "family"
  | "workplace"
  | "apartment"
  | "pet"
  | "elderly"
  | "foreignResident";

type LibraryItem = {
  id: string;
  title: string;
  summary: string;
  category: LibraryCategory[];
  targetUsers: string[];
  sourceName: string;
  sourceUrl: string;
  trustRank: "A" | "B" | "C" | "D";
  estimatedReadMinutes?: number;
  canSaveOffline?: boolean;
  lastCheckedAt?: string;
};
```

## 緊急行動

```ts
type EmergencyAction = {
  id: string;
  title: string;
  summary: string;
  priority: number;
  steps: string[];
  nextActionIds?: string[];
  warning?: string;
  relatedLinks?: {
    label: string;
    url: string;
    sourceName: string;
    trustRank: "A" | "B" | "C" | "D";
  }[];
  offlineAvailable: boolean;
};
```

## localStorageキー案

```text
bousai-navi:saved-areas
bousai-navi:user-profile
bousai-navi:last-selected-prefecture
bousai-navi:last-selected-municipality
bousai-navi:offline-ready
```

## 注意点

- 個人情報を保存しすぎない
- 住所の詳細入力はMVPでは不要
- 自治体単位で止める
- 家族の氏名や電話番号はMVPでは扱わない
- サーバー送信はしない
