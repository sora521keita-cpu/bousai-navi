# 受け入れテストと品質基準

## 全体テスト

### 起動

- アプリが起動する
- 初期画面が表示される
- 重大なコンソールエラーがない

### タブ

- 地域マップタブを開ける
- 防災ライブラリタブを開ける
- 緊急タブを開ける
- 緊急固定ボタンから緊急タブへ移動できる

### 地域マップ

- 都道府県を選べる
- 自治体を選べる
- 自治体詳細が表示される
- 公式リンク枠が表示される
- 仮リンクが仮リンクとして表示される

### 登録エリア

- 自宅として登録できる
- 職場として登録できる
- 実家として登録できる
- よく行く場所として登録できる
- 再読み込み後も保存されている

### 必須リンク

- 東京備蓄ナビ「自分に合った備蓄を調べてみよう」が防災ライブラリに含まれている
- 東京都防災ホームページ「東京くらし防災」・「東京防災」の閲覧が防災ライブラリに含まれている
- 上記2件が信頼度Aとして表示される
- 上記2件が仮リンクではなく必須リンクとして扱われる


- 資料カードが表示される
- カテゴリで絞り込める
- ユーザー属性で絞り込める
- 信頼度ラベルが表示される
- 情報源が表示される

### 緊急

- 緊急行動カードが表示される
- カードを押すと手順が表示される
- 次の行動への導線が表示される
- 外部リンクを開かなくても最低限の手順が読める

### オフライン

- オフライン表示が出る
- 緊急行動カードが読める
- 外部リンクが使えない可能性を明示する

### 共有

- 家族向け共有文を生成できる
- 職場向け共有文を生成できる
- Web Share APIまたはコピー機能が使える

## UX品質基準

### 緊急時UX

- 重要ボタンがすぐ見つかる
- 1画面に情報を詰め込みすぎない
- 行動手順は短文
- 判断が必要な場合は「自治体公式情報を確認」と明示する
- 危険な断定を避ける

### 平時UX

- 自宅・職場・実家を登録しやすい
- 防災資料を探しやすい
- 家族や職場に共有しやすい
- 自治体情報へすぐ飛べる

## セキュリティ・プライバシー

- 位置情報を無断取得しない
- サーバーに個人情報を送らない
- localStorageのみに保存する
- 住所詳細を必須にしない
- 家族の氏名・電話番号をMVPでは扱わない

## 禁止表現

以下のような断定は避ける。

- 「この避難所に行けば安全です」
- 「この情報だけ見れば大丈夫です」
- 「津波の心配はありません」
- 「この地域は安全です」

代わりに以下のように書く。

- 「自治体の最新情報を確認してください」
- 「周囲の状況に応じて安全を優先してください」
- 「海岸・河川の近くでは津波情報を確認してください」
- 「危険を感じる場合は無理に移動しないでください」

## ビルド確認

各段階の最後に以下を実行する。

```bash
npm run build
```

テストがある場合：

```bash
npm test
```

Lintがある場合：

```bash
npm run lint
```

## 完了条件

各段階で、以下を満たしたら完了。

- 実装要件を満たす
- ビルドが通る
- 重大な型エラーがない
- UIがモバイル幅で崩れない
- 緊急タブがすぐ開ける
- オフライン時の最低限動作が確認できる
