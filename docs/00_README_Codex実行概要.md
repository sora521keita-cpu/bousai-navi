# 自治体別パーソナル防災ナビ — Codex開発指示書セット

## 目的

この指示書セットは、Codexに対して「自治体別・自分用 防災ナビ」のMVPを段階的に開発させるためのものです。

このアプリは、既存の地震通知アプリと真正面から競合するものではありません。  
主目的は、災害通知そのものではなく、以下の支援です。

- 自分の自治体・職場・実家などの防災情報を探しやすくする
- 緊急時に迷わず行動できる行動カードを提示する
- 信頼できる公的資料へのリンクを整理する
- 通信不能時でも最低限の行動手順を確認できる
- 家族・職場・自治会に共有できる防災ページの土台を作る

## 推奨する最初の実装方針

最初からネイティブアプリを作らず、まずはWeb/PWAとして実装すること。

理由：

- URL共有しやすい
- QRコード・LINE共有と相性が良い
- ホーム画面追加でアプリ的に使える
- MVP検証が早い
- オフラインキャッシュをService Workerで実装しやすい

## 想定技術スタック

特に指定がない場合、以下を前提とする。

- フロントエンド: React + TypeScript
- ビルド: Vite
- 地図: SVG日本地図または簡易GeoJSON表示
- 状態管理: localStorageから開始
- オフライン: Service Worker
- データ: JSONファイル
- スタイル: CSS Modulesまたは通常CSS
- テスト: 最低限の型チェック・ビルド確認

外部APIやリアルタイム災害通知はMVPでは扱わない。

## 段階構成

- 第1段階: UI骨格と3タブ構成
- 第2段階: 自治体データ・地図選択・お気に入り登録
- 第3段階: 防災ライブラリ・緊急行動カード・オフライン対応
- 第4段階: 共有ページ・品質改善・導入検証

## Codexへの基本方針

Codexには、各段階ファイルを順番に読み込ませて実装させること。

一度に全段階を実装させると、責務が膨らみ、UI・データ・オフライン・共有機能が混線しやすい。  
必ず段階的に進める。

## 実装時の禁止事項

- 災害予測を独自に行わない
- 緊急地震速報を自前実装しない
- 公的機関の情報を改変して断定的に表示しない
- ユーザー位置情報を無断取得しない
- 個人情報をサーバーに送信しない
- 緊急時の行動カードを長文化しない
- 単なるリンク集だけで終わらせない


## 必須収録リンク

防災ライブラリの初期データには、以下2件を必ず含めること。

1. 東京備蓄ナビ「自分に合った備蓄を調べてみよう」
   - URL: https://www.bichiku.metro.tokyo.lg.jp/tool/
   - 用途: ユーザーの世帯構成・住居・ペット有無などに応じた備蓄確認導線
   - 推奨カテゴリ: stockpile, family, pet, elderly
   - 信頼度: A
   - 情報源: 東京都

2. 東京都防災ホームページ「東京くらし防災」・「東京防災」の閲覧
   - URL: https://www.bousai.metro.tokyo.lg.jp/1028036/1028051/index.html
   - 用途: 防災ブック、PDF、電子書籍、多言語版への導線
   - 推奨カテゴリ: earthquake, stockpile, family, apartment, elderly, foreignResident
   - 信頼度: A
   - 情報源: 東京都

この2件は「おすすめ候補」ではなく、MVP初期データに必ず入れる。

## MVPの合格ライン

MVPでは以下ができればよい。

1. 下部3タブがある
   - 地域マップ
   - 防災ライブラリ
   - 緊急
2. 自治体を選択できる
3. 自宅・職場・実家として自治体を保存できる
4. 自治体別の公式リンクを表示できる
5. 緊急行動カードを表示できる
6. 最低限の情報がオフラインでも読める
7. 家族・職場に共有しやすいテキストを生成できる

## 最重要UX原則

緊急時にユーザーへ考えさせすぎない。

- 「今ゆれている」
- 「揺れがおさまった」
- 「避難する」
- 「家族と連絡する」
- 「火災・ガス臭い」
- 「停電・断水」

のように、状況から行動へ直行できるUIにする。
