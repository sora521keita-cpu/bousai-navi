# 第四段階指示ファイル — 共有機能・品質改善・導入検証

## ゴール

アプリを「個人で見る防災情報」から、「家族・職場・自治会で共有できる防災ナビ」に進化させる。

## 共有機能の考え方

防災は個人だけでは完結しない。  
家族、職場、実家、マンション、自治会で共有できることが重要。

MVPではサーバー共有ではなく、共有テキスト生成から始める。

## 共有テキスト生成

登録エリアと緊急連絡導線をもとに、LINEやメールへ貼り付けられるテキストを生成する。

例：

```text
【わが家の防災メモ】

自宅の自治体:
さいたま市

確認する情報:
・自治体公式防災ページ
・ハザードマップ
・避難所・避難場所
・災害用伝言ダイヤル171

地震直後の行動:
1. 頭を守る
2. 揺れがおさまるまで動かない
3. 火元・出口・靴を確認
4. 家族の安否確認
5. 必要なら避難

家族で決めておくこと:
・集合場所
・連絡手段
・避難先
・備蓄場所
```

## Web Share API

可能ならWeb Share APIを使う。  
対応していない環境では、コピー用ボタンを出す。

実装方針：

- `navigator.share` が使える場合は共有
- 使えない場合はクリップボードへコピー
- 失敗した場合はテキストエリアに表示して手動コピー

## 家族・職場モード

共有テンプレートを2種類作る。

### 家族向け

- 自宅
- 実家
- 集合場所
- 171 / web171
- 避難所
- 備蓄
- ペット・高齢家族メモ

### 職場向け

- 職場自治体
- 従業員安否確認
- 顧客対応
- 車両・設備確認
- ライフライン確認
- 帰宅困難時対応
- 本社報告メモ

## 品質改善

### アクセシビリティ

- 文字サイズを小さくしすぎない
- タップ領域を十分大きくする
- 色だけで情報を区別しない
- 緊急ボタンは視認性を高くする
- 重要情報は短文にする

### 信頼性

- 情報源を必ず表示する
- 仮リンクを明示する
- 更新日欄を持たせる
- 公的情報を優先する
- 独自判断を断定的に表示しない

### エラーハンドリング

- リンク未設定
- オフライン
- localStorage使用不可
- 共有API使用不可
- 位置情報拒否

## 位置情報

この段階で現在地判定を実装する場合は、必ず許可制にする。

- 初回に勝手に取得しない
- 「現在地から自治体を探す」を押したときだけ取得する
- 位置情報は端末内処理を原則とする
- サーバー送信しない
- 拒否された場合は自治体検索へ誘導する

MVPでは、現在地から自治体を厳密判定しなくてもよい。  
実装負荷が高い場合は、都道府県または近隣自治体候補表示までで止める。

## 受け入れ条件

- 家族向け共有テキストを生成できる
- 職場向け共有テキストを生成できる
- Web Share APIまたはコピー機能が使える
- 仮リンク・未設定リンクの扱いが明確
- アクセシビリティ上の最低限の配慮がある
- オフライン時にも緊急行動カードが読める
- ビルドが通る

## Codexへの実行指示

```text
第三段階までのアプリに、家族向け・職場向けの共有テキスト生成機能を追加してください。
登録エリア、自治体リンク、緊急行動カード、171/web171などの連絡導線をもとに、LINEやメールへ貼り付けやすい文章を生成してください。
Web Share APIが利用できる場合は共有ボタンで共有し、利用できない場合はクリップボードコピーまたは手動コピー用テキストエリアを表示してください。
あわせてアクセシビリティ、仮リンク表示、オフライン時の表示、localStorage失敗時のエラーハンドリングを改善してください。
実装後、ビルド確認を行い、エラーがあれば修正してください。
```
