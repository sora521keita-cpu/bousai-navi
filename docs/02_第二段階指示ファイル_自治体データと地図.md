# 第二段階指示ファイル — 自治体データ・地図選択・お気に入り登録

## ゴール

第一段階のUI骨格に、自治体選択・自治体情報表示・登録エリア保存機能を追加する。

この段階では、全国完全網羅を目指さない。  
まずは首都圏のサンプル自治体を対象にする。

## 対象範囲

MVPでは以下の都県から開始する。

- 東京都
- 埼玉県
- 千葉県
- 神奈川県

最初は各都県3〜5自治体程度のサンプルでよい。

例：

東京都:
- 新宿区
- 世田谷区
- 江東区

埼玉県:
- さいたま市
- 川口市
- 北本市
- 越谷市

千葉県:
- 千葉市
- 船橋市
- 市川市

神奈川県:
- 横浜市
- 川崎市
- 相模原市

## データ設計

`src/data/municipalities.ts` または `src/data/municipalities.json` を作る。

自治体データの基本形：

```ts
type Municipality = {
  id: string;
  prefecture: string;
  name: string;
  municipalityCode?: string;
  officialWebsiteUrl?: string;
  disasterPageUrl?: string;
  hazardMapUrl?: string;
  evacuationInfoUrl?: string;
  lifelineInfoUrl?: string;
  notes?: string;
};
```

## 重要な注意

この段階でURLが未確認の場合は、無理に架空URLを入れない。  
仮URLの場合は必ず `isPlaceholder: true` のようなフラグを持たせる。

推奨型：

```ts
type SourceLink = {
  label: string;
  url: string;
  sourceType: "official" | "public" | "infrastructure" | "placeholder";
  isPlaceholder?: boolean;
};

type Municipality = {
  id: string;
  prefecture: string;
  name: string;
  municipalityCode?: string;
  links: SourceLink[];
  notes?: string;
};
```

## 地域マップタブの表示要件

### 上部

- 現在地から探す
  - この段階ではボタンのみ
  - 押すと「現在地判定は今後実装予定」と表示
- 自治体名検索
- 都道府県選択
- お気に入り登録エリア

### 自治体一覧

都道府県を選択すると、自治体リストを表示する。

自治体を押すと詳細画面または詳細パネルを表示する。

### 自治体詳細

表示内容：

- 自治体名
- 都道府県
- 公式サイト
- 防災ページ
- ハザードマップ
- 避難所・避難場所情報
- ライフライン情報
- 自宅に登録
- 職場に登録
- 実家に登録
- よく行く場所に登録

## お気に入り登録

localStorageに保存する。

保存項目：

```ts
type SavedAreaRole = "home" | "work" | "familyHome" | "frequent";

type SavedArea = {
  role: SavedAreaRole;
  municipalityId: string;
  label?: string;
};
```

画面上部の「登録エリア」に反映する。

## UX要件

- 自治体を選んだ後、すぐ公式情報へ飛べる
- 登録エリアは常に見つけやすい
- 自宅・職場・実家を区別できる
- URL未確認のリンクはユーザーに誤解させない
- 「これは仮リンクです」表示を出す

## 受け入れ条件

- 都道府県を選べる
- 自治体を選べる
- 自治体詳細が見られる
- 自宅・職場・実家として登録できる
- localStorageに保存される
- 再読み込み後も登録が残る
- 仮URLと実URLを区別して表示できる
- TypeScriptエラーがない
- ビルドが通る
