# 第三段階指示ファイル — 防災ライブラリ・緊急行動カード・オフライン対応

## ゴール

防災ライブラリと緊急タブを、単なるリンク集ではなく「行動支援」に進化させる。  
さらに、最低限の情報をオフラインでも読めるようにする。

## 防災ライブラリの考え方

防災ライブラリは「資料置き場」ではない。  
ユーザー属性・状況に応じて、読むべき資料を見つけやすくする場所である。

## 防災資料データ設計

`src/data/libraryItems.ts` を作る。

```ts
type TrustRank = "A" | "B" | "C" | "D";

type LibraryCategory =
  | "earthquake"
  | "tsunami"
  | "flood"
  | "landslide"
  | "lifeline"
  | "stockpile"
  | "family"
  | "workplace"
  | "apartment"
  | "pet"
  | "elderly"
  | "foreignResident";

type LibraryItem = {
  id: string;
  title: string;
  summary: string;
  category: LibraryCategory[];
  targetUsers: string[];
  sourceName: string;
  sourceUrl: string;
  trustRank: TrustRank;
  estimatedReadMinutes?: number;
  canSaveOffline?: boolean;
};
```

## 信頼度ラベル

信頼度は以下のルールにする。

- A: 国、気象庁、消防庁、自治体、NTTなどの一次情報
- B: 公的外郭団体、自治体関連団体、公共性の高い団体
- C: 報道機関、民間企業の解説
- D: 個人ブログ、SNS、出所不明情報

MVPではAとBを優先表示する。


## 必須収録リンク

防災ライブラリの初期データには、以下2件を必ず含めること。

1. 東京備蓄ナビ「自分に合った備蓄を調べてみよう」
   - URL: https://www.bichiku.metro.tokyo.lg.jp/tool/
   - 用途: ユーザーの世帯構成・住居・ペット有無などに応じた備蓄確認導線
   - 推奨カテゴリ: stockpile, family, pet, elderly
   - 信頼度: A
   - 情報源: 東京都

2. 東京都防災ホームページ「東京くらし防災」・「東京防災」の閲覧
   - URL: https://www.bousai.metro.tokyo.lg.jp/1028036/1028051/index.html
   - 用途: 防災ブック、PDF、電子書籍、多言語版への導線
   - 推奨カテゴリ: earthquake, stockpile, family, apartment, elderly, foreignResident
   - 信頼度: A
   - 情報源: 東京都

この2件は「おすすめ候補」ではなく、MVP初期データに必ず入れる。

## 緊急タブの考え方

緊急タブでは、外部リンクより先にアプリ内の行動手順を表示する。  
通信不能時でも読める必要がある。

## 緊急行動カードデータ設計

`src/data/emergencyActions.ts` を作る。

```ts
type EmergencyAction = {
  id: string;
  title: string;
  summary: string;
  priority: number;
  steps: string[];
  nextActionIds?: string[];
  warning?: string;
  relatedLinks?: {
    label: string;
    url: string;
    sourceName: string;
    trustRank: TrustRank;
  }[];
  offlineAvailable: boolean;
};
```

## 初期カード

最低限以下を実装する。

### 今ゆれている

手順：

1. 頭を守る
2. 窓・棚・照明から離れる
3. 机の下や安全な場所に移動する
4. 無理に火を消しに行かない
5. 揺れがおさまるまで動かない

次の行動：

- 揺れがおさまった

### 揺れがおさまった

手順：

1. 自分と周囲のけがを確認する
2. 火元を確認する
3. ガス臭い場合は火を使わない
4. 靴を履く
5. 出口を確保する
6. 家族・同居人の安否を確認する
7. 必要なら避難する

次の行動：

- 火災・ガス臭い
- 家族と連絡
- 避難したい

### 火災・ガス臭い

手順：

1. 火が出ている場合は安全を最優先する
2. 初期消火できない場合はすぐ離れる
3. ガス臭い場合は火気・スイッチ・換気扇を使わない
4. 可能なら窓を開ける
5. 安全な場所から119番へ連絡する

### 家族と連絡したい

手順：

1. 電話がつながらない前提で考える
2. 災害用伝言ダイヤル171を使う
3. web171を使う
4. SNSやメッセージアプリは短文にする
5. 集合場所を事前に決めている場合は確認する

### 避難したい

手順：

1. 自治体の避難情報を確認する
2. 避難所と避難場所の違いに注意する
3. ブレーカー・ガス元栓などを可能な範囲で確認する
4. 最低限の持ち物を持つ
5. 建物・ブロック塀・河川・海岸に注意して移動する

## オフライン対応

Service Workerを導入する。  
MVPでは以下をキャッシュする。

- アプリ本体
- 緊急行動カード
- 防災ライブラリのメタ情報
- 登録自治体情報
- localStorage上の登録エリア

注意：

- 外部リンク先そのものはキャッシュしなくてよい
- 外部リンクが開けない場合に備え、アプリ内の行動カードを優先する

## オフライン表示

ネットワークがない場合は、画面上部に表示する。

```text
オフラインです。外部リンクは開けない可能性がありますが、緊急行動カードは確認できます。
```

## 受け入れ条件

- 防災ライブラリに信頼度ラベルが出る
- カテゴリ・属性で資料を絞り込める
- ユーザー属性がlocalStorageに保存される
- 緊急行動カードに手順と次の行動が表示される
- オフライン時に外部リンク依存にならない
- Service Workerが導入されている
- ビルドが通る

## Codexへの実行指示

```text
第二段階までのアプリに、防災ライブラリのカテゴリ・信頼度ラベル・ユーザー属性フィルターを追加してください。
また、緊急タブを外部リンク集ではなく、アプリ内で完結する状況別行動カードとして実装してください。
各行動カードには手順、注意文、次に取る行動への導線を持たせてください。
Service Workerを導入し、アプリ本体と緊急行動カードを最低限オフラインでも確認できるようにしてください。
外部リンクは補助扱いにし、通信不能時でも最低限の緊急手順が読める設計にしてください。
実装後、ビルド確認を行い、エラーがあれば修正してください。
```
