# 第一段階指示ファイル — UI骨格と3タブ構成

## ゴール

「自治体別パーソナル防災ナビ」の最小UIを構築する。

この段階では、データ連携や高度な地図表示は不要。  
まず、ユーザーがアプリ全体の構造を理解できる状態を作る。

## アプリ名

仮称：

- わが家の防災ナビ
- 自治体別 防災ナビ
- パーソナル防災ナビ

実装上は `BousaiNavi` としてよい。

## 画面構成

下部タブを3つ作る。

1. 地域マップ
2. 防災ライブラリ
3. 緊急

ただし、緊急タブは常にアクセスできるよう、画面上部または右下にも固定ボタンを置く。

## レイアウト要件

### 共通ヘッダー

表示項目：

- アプリ名
- 短い説明文
- 緊急ボタン

例：

```text
わが家の防災ナビ
自治体別の防災情報と緊急時の行動をまとめて確認できます。
[緊急]
```

### ホーム相当の表示

最初の画面は「地域マップ」タブでよい。

ただし、上部に以下を表示する。

- 登録エリア
  - 自宅: 未登録
  - 職場: 未登録
  - 実家: 未登録
- 「自治体を探す」ボタン
- 「緊急時の行動を見る」ボタン

## タブ1: 地域マップ

この段階では本格地図は不要。  
日本地図のプレースホルダーを置き、都道府県選択のUIを作る。

表示内容：

- 現在地から探す
- 日本地図から探す
- 都道府県リスト
- お気に入り自治体
  - 自宅
  - 職場
  - 実家
  - よく行く場所

都道府県リストは、最初はダミーでよい。

例：

- 東京都
- 埼玉県
- 千葉県
- 神奈川県

## タブ2: 防災ライブラリ

「リンク集」ではなく「防災ライブラリ」と表記する。

カード形式で表示する。

初期カード例：

1. 地震にあったら
   - 対象: 全員
   - 種別: 緊急行動
   - 情報源: 消防庁などの公的資料を想定
2. 家庭の備蓄
   - 対象: 一人暮らし・家族
   - 種別: 平時の備え
3. 家族との安否確認
   - 対象: 家族・職場
   - 種別: 連絡手段
4. みんなで防災
   - 対象: 学校・職場・自治会
   - 種別: 防災学習

この段階ではリンクURLは仮でよい。  
ただしデータ構造は後で差し替えられるようにする。

## タブ3: 緊急

緊急タブは行動カード中心にする。  
外部リンク中心にしない。

表示カード：

- 今ゆれている
- 揺れがおさまった
- 外にいる
- 車に乗っている
- 海・川の近くにいる
- 火災・ガス臭い
- 停電・断水
- 家族と連絡したい
- 避難したい
- 助けを呼びたい

各カードを押すと、簡単な手順を表示する。

例：

```text
今ゆれている

1. 頭を守る
2. 窓・棚・照明から離れる
3. 机の下や安全な場所に移動
4. 無理に火を消しに行かない
5. 揺れがおさまるまで動かない
```

## 実装要件

- React + TypeScriptで実装
- コンポーネントを分割する
  - App
  - Header
  - BottomTabs
  - RegionMapTab
  - LibraryTab
  - EmergencyTab
  - EmergencyCard
  - LibraryCard
- CSSは見通しよく整理する
- モバイルファーストで作る
- 片手操作を意識する
- 文字サイズは小さくしすぎない

## 受け入れ条件

- アプリが起動する
- 3タブを切り替えられる
- 緊急ボタンから緊急タブへ移動できる
- 防災ライブラリのカードが表示される
- 緊急行動カードを押すと手順が見られる
- TypeScriptエラーがない
- ビルドが通る

以下の方針で実装してください。

```text
React + TypeScript + Viteで、自治体別パーソナル防災ナビの第一段階MVPを実装してください。

実装後、npm buildまたは同等のビルド確認を行い、エラーがあれば修正してください。
```
